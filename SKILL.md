# 金融运营技能工厂

## 路由入口

**触发条件：** 用户提到基金运营、活动策划、用户增长、文案撰写、社区舆情、数据分析、合规审查等互金运营场景。

**路由决策：**

```
用户输入
    │
    ├─ "启动运营周期" / "新季度规划" → 全流程串行
    │    1. 燃冰定方向
    │    2. 数据线分析→出策略
    │    3. 运营线执行
    │    4. 合规线审查
    │    5. 数据线复盘
    │    6. 燃冰调方向
    │
    ├─ "写文案" / "做内容" → 运营线→合规线串行
    │    1. 运营线产出内容
    │    2. 合规线审查→放行/驳回
    │
    ├─ "设计活动" / "用户增长" → 运营线→合规线串行
    │    1. 运营线设计活动方案
    │    2. 合规线审查→放行/驳回
    │
    ├─ "分析数据" / "复盘效果" → 数据线独立
    │    1. 数据线分析→出报告
    │
    ├─ "监控舆情" / "社区动态" → 数据线独立
    │    1. 数据线采集分析→出简报
    │
    ├─ "检查合规" / "这个能发吗" → 合规线独立
    │    1. 合规线审查→放行/驳回
    │
    └─ 其他 → 匹配最相关的角色，否则报"需要更明确的需求"
```

## 核心规则

0. **原子知识库为所有产出的前置输入**：任何角色开始工作前，必须先读取 `knowledge/atoms-knowledge-base.md` 中对应场景的模块内容。这是燃冰的真实经验，不可跳过。

1. **串行不并行**：所有多角色任务严格按 角色A→角色B→角色C 顺序执行
2. **上一步的产出即下一步的输入**：不重新理解、不重复分析
3. **合规线永远在发布前最后一步**：不提前、不跳过
4. **燃冰只在"定方向"和"拍板"时介入**：执行过程不需要他
5. **每条路必须过对应闸**：运营产出过合规闸，数据产出过数据闸，阶段转换过验证闸
