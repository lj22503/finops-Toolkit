# 数据导航与归因

## Overview

数据是运营决策的基础，必须会看、会分析、会归因。

## When to Use

- 日常数据监控时
- 活动效果分析时
- 发现数据异常时

## Core Skills

### 看数据

**关键指标：**

| 维度 | 指标 | 说明 |
|------|------|------|
| 流量 | PV/UV/跳出率 | 内容曝光 |
| 转化 | 注册率/开户率 | 获客效率 |
| 留存 | D7/D30留存率 | 用户粘性 |
| 收入 | 持有率/复购率 | 商业价值 |

### 归因分析

**数据下滑时，问：**

1. 是文案问题？ → 点击率高但转化低
2. 是活动奖励不够？ → 参与高但转化低
3. 是渠道流量不精准？ → 整体漏斗都低

**分析方法：**
- 漏斗分析（每步转化率）
- 同期群分析（Cohort）
- 分维度拆解（渠道/人群/时间）

### 找捷径

- 通过数据发现"最佳路径"（高效渠道/优质内容）
- 识别高价值用户特征
- 找到低投入高产出点

## Data Quality Rules

| 要求 | 说明 |
|------|------|
| 来源可查 | 所有数据必须有来源 |
| 时间明确 | 标注统计口径和截止日期 |
| 分布完整 | 不只报平均数，标注中位数/分位数 |
| 异常标记 | 异常数据必须标注"数据异常，原因待查" |

## Gate Rule

涉及数据判断的产出，过数据闸。

## Anti-Patterns

- 看平均数不看分布 → 被极端值误导
- 数据幻觉 → 关联不等于因果
- 不标注数据异常 → 把异常当常规
