# 生态化做局能力

## Overview

不把用户当"流量"，而是生态的一部分。让用户和平台互相成就。

## When to Use

- 设计用户激励体系时
- 规划社区功能时
- 设计自增长机制时

## Core Concept

**好的生态 = 用户得到价值 + 平台得到增长**

不是"薅用户"，而是"和用户一起赢"。

## Application

### 用户身份体系

**示例：**
- "基金问答达人"身份
- 持有天数徽章
- 社区贡献积分

**逻辑：**
- 用户获得荣誉/归属感
- 平台得到UGC内容和自传播

### 互助体系

**示例：**
- 老用户带新用户（社区信任背书）
- 用户回答用户问题（降低客服压力）
- 投资心得分享（UGC内容）

### 自传播设计

**示例：**
- 收益分享（晒收益但合规）
- 邀请好友（双方得利）
- 内容转发（品牌传播）

## Design Principles

| 原则 | 说明 |
|------|------|
| 激励真实价值 | 不是薅羊毛，是价值交换 |
| 低门槛参与 | 任何用户都能参与 |
| 可持续 | 不是一次性补贴，是长期机制 |

## Anti-Patterns

- 把用户当流量 → 一次性的薅羊毛
- 激励虚假行为 → 用户刷单/刷量
- 机制不可持续 → 补贴停了用户也跑了
