# AARRR 漏斗实操

## Overview

用户增长核心框架：Acquisition → Activation → Retention → Revenue → Referral

## When to Use

- 设计用户增长策略时
- 分析用户生命周期时
- 规划促活/留存活动时

## Five Modules

### Acquisition（开源·拉新）

**核心问题：** 如何让新用户进来？

**策略：**
- 注册送福利（新用户专项）
- 老带新裂变（社交传播）
- 渠道投放（内容/广告）

**金融适配注意：**
- 不能承诺收益作为拉新钩子
- 可以用"免费体验金"、"学习资料"作为拉新激励

### Activation（激活·体验）

**核心问题：** 如何让用户完成关键动作？

**策略：**
- 识别"aha moment"（首次体验到产品价值）
- 缩短首次体验路径
- 新用户引导（功能介绍+激励）

**金融适配注意：**
- 首次购买决策门槛高，需要更多信任建立

### Retention（节流·留存）

**核心问题：** 如何让用户持续使用？

**策略：**
- 用户流失预警（7天未登录等）
- 定期触达（但控制频率，防过度打扰）
- 签到/积分体系

**金融适配注意：**
- 持仓用户需要"持有成就感"
- 产品设计用"持有收益可视化"

### Revenue（转付费）

**核心问题：** 如何引导用户首次/持续投资？

**策略：**
- 定投计划推荐
- 大额券发放引导首投
- 复购激励

**合规注意：**
- 不能承诺收益
- 风险提示必须完整

### Referral（推荐·传播）

**核心问题：** 如何让用户主动分享/拉人？

**策略：**
- 邀请好友得奖励（双方都要合规）
- 用户荣誉体系（"基金问答达人"等）
- UGC 内容引导

## Gate Rule

涉及数据判断的产出，必须过数据闸。

## Anti-Patterns

- 只关注拉新，忽视留存 → 漏斗流失严重
- 用短期激励掩盖产品价值缺失
