## Hard Gate: 验证闸

**触发条件：** 每个Phase完成、每个skill或角色文件初稿完成时

**检查清单：**

### 可量化
- [ ] 方案有明确的成功指标（KPI/OKR）
- [ ] 效益预测有计算依据，不是拍脑袋
- [ ] 对比方案有基准线（control group / before state）
- [ ] 指标与业务目标直接关联（不偷换概念）

### 可验证
- [ ] 方案有验证方法（A/B测试/追踪埋点/数据看板）
- [ ] 验证周期明确（多久能看出效果）
- [ ] 验证成本低于方案收益预期
- [ ] 失败标准提前定义（什么情况算失败）

### 可执行
- [ ] 方案有明确的执行步骤和负责人
- [ ] 资源需求已标注（人力/预算/工具）
- [ ] 风险预案存在（如果效果不佳怎么调整）
- [ ] 有时间节点（什么时候完成什么）

### 文档质量
- [ ] AI 能根据描述独立执行（不依赖人工解释）
- [ ] 触发条件清晰（什么场景下AI应该调用该skill）
- [ ] 禁止词：无"众所周知""显然""我们可以"

**强度：** HARD

**失败处理：**
- ❌ 指标模糊 → "提升转化率"不通过，需改为"将详情页到确认页的转化率从12%提升至15%"
- ❌ 无验证方法 → 补A/B测试方案或数据追踪方案
- ❌ 文档不可执行 → 重写为AI可直接使用的格式
- ✅ 全部通过 → 进入整合层

**防绕过原则：**
| 绕过话术 | 反击 |
|---------|------|
| "这个方案效果明显，不需要验证" | 所有方案都需要验证 |
| "指标后面再定" | 指标是方案的一部分，不是后加的 |
| "AI先产出后面人工改" | 产出的标准是AI可直接使用，不是草稿 |
