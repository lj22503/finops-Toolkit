## Hard Gate: 数据闸

**触发条件：** 任何skill、角色定义、方案输出中涉及数据/数据判断时

**检查清单：**

### 数据来源（必须全部通过）
- [ ] 每个数据/统计结论都标注了明确来源
- [ ] "待更新""待确认""待填充"标记 → ❌ 禁止
- [ ] 模拟/估算数据标注了"估算"和估算方法
- [ ] 外部数据标注了查询日期和API名称

### 数据质量
- [ ] 数据未过期（行情数据 < 1交易日，宏观数据 < 7天）
- [ ] 数据口径统一（不同来源的同类数据需交叉验证）
- [ ] 不存在明显的数量级/单位错误
- [ ] 对比分析中的数据基准合理

### 判断约束
- [ ] 不是"我觉得"，而是"数据表明"
- [ ] 区分"可验证事实"和"推测意见"
- [ ] 涉及预测时标注关键假设和置信度
- [ ] 不将短期波动解释为长期趋势

**强度：** HARD

**失败处理：**
- ❌ "待更新" → 直接退回，必须填充或删掉
- ❌ 无来源数据 → 查找来源或标注"估算"，否则退回
- ❌ 过时数据 → 重新拉取或标注时效性
- ✅ 全部通过 → 放行

**防绕过原则：**
| 绕过话术 | 反击 |
|---------|------|
| "数据之后补" | 之后不会补。现在就要。 |
| "这是常识" | 常识也要有来源。写出来。 |
| "API暂时拿不到" | 标注"估算"并注明数据缺口 |
