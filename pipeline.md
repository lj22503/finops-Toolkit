# 金融运营技能工厂 · 执行管道

**三层联通：** 知识层(skills/) → 执行层(roles/) → 纪律层(gates/)

---

## 角色架构（3+1）

```
        ┌──────────────────────────────────────┐
        │   燃冰（运营决策者）                    │
        │   制定方向 · 分配资源 · 裁决分歧 · 拍板  │
        └──────────────────┬───────────────────┘
                           │ 方向/目标/预算
                           ↓
        ┌──────────────────────────────────────┐
        │   数据线                              │
        │   看数据→分析→出策略建议→追踪验证       │
        │   （大脑）                             │
        └──────────┬────────────┬──────────────┘
                   │            │
           策略建议 ↓            ↘ 效果数据
        ┌────────────────────┐  ┌────────────────────┐
        │   运营线             │  │   合规线             │
        │   内容+活动+触达     │  │   审查+把关+风控      │
        │   （手脚）           │←│   （门禁）            │
        └────────────────────┘  └────────────────────┘
                   │                      ↑
                   └──── 合规审查 ─────────┘
```

**核心原则：**
- **3个Agent角色**：运营线、数据线、合规线
- **+1个人类角色**：燃冰（运营决策者）— 不做成Agent，不设独立Prompt
- 燃冰只做三件事：**定方向、分资源、拍板**
- Agent之间不设"负责人"角色（没有层层汇报，只有对当前目标负责）

---

## 管道各阶段

```
Stage 1: 定向（燃冰→全团队）
  触发：运营周期开始（月度/季度/突发事件）
  输入：公司目标 + 市场动向 + 存量数据
  执行：燃冰输出方向与目标
  输出：运营方向文档（含关键指标和预算）
  过闸：验证闸（方向可量化？目标可衡量？）

Stage 2: 分析（数据线→运营线）
  触发：收到方向
  执行：数据线基于方向出策略建议、用户分层、目标拆解
  输出：策略方案（含A/B测试假设）
  过闸：合规闸+数据闸+验证闸

Stage 3: 执行（运营线）
  触发：收到数据线的策略建议 + 燃冰确认
  执行：内容生产 / 活动设计 / 用户触达，一气呵成
  输出：内容稿件 / 活动方案 / 触达计划
  过闸：合规闸（每份产出必过）

Stage 4: 把关（合规线）
  触发：运营线提交产出
  执行：逐条过合规检查清单
  输出：合规审查报告 / 审批通过或驳回
  驳回流程：退回运营线修改 → 重新提交
  过闸：合规闸（一票否决）

Stage 5: 复盘（数据线→全团队）
  触发：运营活动/周期结束
  执行：数据线做效果复盘和归因分析
  输出：复盘报告（含数据验证）
  过闸：数据闸+验证闸

Stage 6: 迭代（数据线+燃冰→下一周期）
  触发：复盘报告提交
  执行：燃冰Review → 调整下一周期方向
  输出：迭代后的运营方向
  过闸：验证闸
```

---

## 节点间数据传递规范

```
同一运营周期内：
  方向文档（Stage 1输出）→ 全团队输入锚点
  策略方案（Stage 2输出）→ 运营线的执行依据
  合规报告（Stage 4输出）→ 上线/下线的决策依据
  复盘报告（Stage 5输出）→ 下一周期策略调整的依据

异常流程：
  合规闸不通过 → 退回运营线修改 → 重新提交
  数据异常 → Stage 5卡住直到数据确认
  方向变更 → 数据线重新出策略方案 → 运营线调整执行
  跨线冲突 → 燃冰裁决（运营线 vs 数据线的"资源优先"冲突）
```

## 角色职责速览

| 角色 | 一句话描述 | 输出给谁 | 谁的输入 |
|------|-----------|---------|---------|
| 燃冰 | 定方向、分资源、拍板 | 全团队 | 数据线复盘 |
| 🟢 运营线 | 全栈触达：内容+活动+渠道 | 用户（直接触达） | 数据线策略 + 合规线审查 |
| 🔵 数据线 | 大脑：分析+策略+验证 | 运营线 + 燃冰 | 运营线效果数据 |
| 🔴 合规线 | 门禁：审查+把关+风控 | 运营线（驳回/放行） | 运营线产出 |

## 3道闸嵌入点

```
                    ┌──→ 合规闸 ──→ 退回修改
                    │    (每份产出)
各线产出 ──────────┤
                    │
                    └──→ 数据闸 ──→ 退回修改
                         (有数据判断的产出)

Stage 1-6 流转过程中：
  阶段转换 → 验证闸（检查产出完整性+可执行性）
  
  终验（Stage 5→6）→ 3闸全过才允许迭代
```
